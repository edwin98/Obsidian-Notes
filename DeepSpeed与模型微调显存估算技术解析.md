# DeepSpeed与模型微调显存估算技术解析

在大型语言模型 (LLM) 的训练和微调过程中，显存 (VRAM) 往往是最大的瓶颈。微软提出的 DeepSpeed 框架通过核心技术 **ZeRO (Zero Redundancy Optimizer)** 有效地打破了单卡显存的限制。本文将详细介DeepSpeed的三种训练模型（划分）方式，并结合各种微调手段（全参数、LoRA、QLoRA）进行显存占用的详细估算。

## 1. 大模型训练显存占用拆解

在了解 DeepSpeed 之前，首先我们需要知道训练一个模型，显存到底被谁吃掉了。
假设模型总参数量为 $\Phi$。目前主流的训练采用**混合精度训练 (Mixed Precision Training)**：前向反向传播使用 fp16/bf16 计算，而在优化器更新参数时使用 fp32 保障精度。

在一个典型的混合精度训练中，不论是预训练还是全参数微调，主要显存占用分为两大类：**静态显存**（模型状态）和 **动态显存**（激活值等）。

### 1.1 静态显存（Model States）
在使用 Adam 优化器的情况下，静态显存可以分为以下几个部分：
1. **模型参数 (Parameters)**：权重参数参与前向和反向传播计算，使用 16-bit (fp16/bf16) 存储。占用 **$2\Phi$ 字节**。
2. **梯度 (Gradients)**：反向传播计算出的梯度，使用 16-bit 存储。占用 **$2\Phi$ 字节**。
3. **优化器状态 (Optimizer States)**：混合精度训练中，由于参数更新的累加值较小，如果使用 16-bit 更新会导致严重下溢。因此 Adam 优化器需要保存：
   - FP32 的参数备份 (Master Weights)：占用 $4\Phi$ 字节。
   - FP32 的 Momentum (一阶动量)：占用 $4\Phi$ 字节。
   - FP32 的 Variance (二阶动量)：占用 $4\Phi$ 字节。
   - 共计 **$12\Phi$ 字节**。

**结论概括：**
全参数训练下，总静态显存占用为 $2\Phi + 2\Phi + 12\Phi = 16\Phi$ 字节。
*案例*：训练一个 LLaMA-7B 模型 ($\Phi=7\times 10^9$)，至少需要 $16 \times 7 = 112$ GB 的静态显存。即便是一张 80G 的 A100 也无法单卡装下！

### 1.2 动态显存
主要指前向传播时产生、用于反向传播求导的**激活值 (Activations)**。
决定激活值大小的因素包括：Batch Size, Sequence Length (上下文长度), Hidden Size (特征维度) 等。
- 可以使用 **Gradient Checkpointing (梯度检查点)** 技术，用计算时间换取空间，大幅度降低激活值的显存开销。

## 2. DeepSpeed 的三种训练方式 (ZeRO的三个阶段)

面对 $16\Phi$ 的庞大显存需求，传统的**数据并行 (Data Parallelism, DP)** 每张显卡都会拷贝一份完整的 Model States，存在极大的显存冗余。
**ZeRO (Zero Redundancy Optimizer)** 的核心思想就是：去冗余。保留数据并行的通信效率，同时通过对 Model States 在多张卡上进行切分，达到类似张量并行 (TP) 的显存节省效果。假设我们使用 $N$ 张显卡。

### 📌 ZeRO-Stage 1 (ZeRO-1)：优化器状态划分
- **原理**：将占用最大的**优化器状态** ($12\Phi$) 切分成 $N$ 份，每张卡只维护自己那部分的优化器状态。
- **显存消耗**：$2\Phi$ (参数) + $2\Phi$ (梯度) + $\frac{12\Phi}{N}$ (优化器状态)。
- **效果**：在不增加通信开销的情况下，显存可节约至原来的 $\frac{1}{4}$ 到 $\frac{1}{3}$ 左右。目前大多全参训练至少默认开启 ZeRO-1。

### 📌 ZeRO-Stage 2 (ZeRO-2)：梯度划分
- **原理**：在 ZeRO-1 的基础上，进一步将**梯度** ($2\Phi$) 也切分成 $N$ 份。每张卡只保留对应切分块的梯度进行独立更新。
- **显存消耗**：$2\Phi$ (参数) + $\frac{2\Phi + 12\Phi}{N}$ (梯度+优化器状态)。
- **效果**：相比 DP，多卡通信量相同，但静态显存可以节约一半以上。

### 📌 ZeRO-Stage 3 (ZeRO-3)：参数划分
- **原理**：在 ZeRO-2 的基础上，将**模型参数** ($2\Phi$) 也切分成 $N$ 份。这意味着每张卡在非计算时刻只持有一小部分参数！当网络的一层需要进行前向或者反向计算时，该卡通过 All-Gather 通信从别的卡那里获取这一层的完整参数；计算完毕后立马丢弃 (释放显存)。
- **显存消耗**：$\frac{16\Phi}{N}$。所有静态显存被 $N$ 张卡完美平摊。
- **效果**：极大地扩展了可训练模型的规模参数。只要显卡足够多，就能放得下千亿级甚至万亿级参数大模型。（代价是增加了近 1.5 倍的通信开销）

> **💡 ZeRO-Offload (额外加成)**：
> 除了将显存在卡间切割，DeepSpeed 还支持将优化器状态和计算**卸载 (Offload) 到 CPU 内存，甚至是 NVMe SSD 固态硬盘**上。通过牺牲部分 PCIe 传输和 CPU 计算时间，让单张低配显卡也能训练大模型。

## 3. 不同微调方式的显存估算与技术细节

这里我们将比较：全参数微调 (Full Finetuning)、LoRA 和 QLoRA。以 **7B（70亿参数）** 模型、使用 **单卡** 或 **8卡 (N=8)** 为例，感受显存的变化。

### 3.1 全参数微调 (Full Parameter Fine-Tuning)
全参微调就是继续上述预训练的过程，更新所有的 $\Phi$ 个参数。
- **静态显存公式**：全量 $16\Phi$ (112GB)。
- **如果不使用 DeepSpeed**：单卡 OOM，因为没有任何单卡能放下 112GB + 动态激活。
- **使用单卡 + ZeRO-Offload CPU**：参数和梯度保留在 GPU ($4\Phi = 28$GB)，优化器状态 ($12\Phi = 84$GB) 放到 CPU。此时 GPU 静态显存 28GB，只要系统内存 $>84$GB 即可。加上动态激活值，24G 显存 (如 3090/4090) 依旧吃力，A6000 (48G) 或 A800 (80G) 则毫无压力。
- **使用 8 张卡 + ZeRO-3**：每张卡静态显存为 $\frac{112}{8} = 14$GB。即便算上 Batch 的激活值占用，非常适合在具有多张 24GB/32GB 显存显卡的服务器上进行。

### 3.2 高效参数微调：LoRA (Low-Rank Adaptation)
LoRA 的原理是**冻结原始基座模型**的所有参数，通过在各层的注意力矩阵旁边旁路增加两个低秩矩阵 $A$ 和 $B$ ($W = W_0 + \Delta W = W_0 + BA$)。
假设新引入的参数量非常小，记为 $\theta$ (通常仅为原模型参数的一小部分，例如 $\theta = 0.002\Phi$，对 7B 模型来说也就是两千万参数左右)。

**显存拆解：**
1. **主模型参数**：完全冻结，不需要 fp32 备份，不产生梯度。由于是 fp16/bf16，依然需要长期存在 GPU 显存用于前向计算。占用 **$2\Phi$ 字节** (约 14 GB)。
2. **LoRA 参数**：需要参与训练，涉及 fp16 参数 ($2\theta$)、fp16 梯度 ($2\theta$)、fp32 优化器状态 ($12\theta$)。因为 $\theta$ 极小，这部分大约只有百兆 MB 级别，几乎可以忽略不计。
- **最终静态显存**：几乎等同于主模型的单纯加载开销，即约 **14GB**。
- **技术细节**：因为绝大部分显存被主模型占据，而主模型又不参与更新，**ZeRO 系列对于 LoRA 训练能够省下的显存非常有限**。如果使用 3090/4090 (24GB) 显卡进行微调，LoRA 使得单卡训练 7B 成为可能 (14GB 静态 + 余下10GB足够分配给激活值)。

### 3.3 极致压缩：QLoRA (Quantized LoRA)
QLoRA 是在 LoRA 的基础上，结合了模型量化技术（通常是 4-bit 量化，如 NF4格式），进一步把主模型的加载显存压榨到了极致。

**显存拆解：**
1. **主模型参数**：从 16-bit 降级为 4-bit。占用从 $2\Phi$ 下降至 **$0.5\Phi$ 字节**。对 7B 模型而言，主模型显存占用直接缩减到 **3.5 GB**！
2. **LoRA 参数**：依然保持 16-bit 和 32-bit 状态，参与训练，体积依旧微小。
3. **Double Quantization (双重量化)**：QLoRA 会对量化常数再次进行量化，每个参数额外能节省 0.37 bit。
4. **Paged Optimizer (分页优化器)**：QLoRA 巧妙利用了 NVIDIA 的统一内存特性。如果 GPU 显存用量超过显存物理上线，它会自动将优化器状态转移至 CPU RAM，防止 OOM 导致的崩溃。

- **最终静态显存**：约 3.5GB ~ 4GB。
- **技术细节**：配合 QLoRA，不仅能在单张 24G 消费级显卡上训练 7B，甚至只要调低一些 Batch Size，单卡微调 13B / 14B 模型 (约 7~8GB 的基础静态显存) 也完全不在话下。计算时，4-bit 参数会临时解压到 16-bit 进行乘法运算，计算完即可丢弃。

## 总结

| 微调策略 | 参数状态变化 | 主力优化工具 | 单卡静态显存评估 (LLaMA-7B为例) | 使用场景 |
| ---- | ---- | ---- | ---- | ---- |
| **全参数微调** | 动全部参数，梯度全回传 | DeepSpeed ZeRO-2 / ZeRO-3 | 约 112 GB (不含激活) | 资源极其丰富；任务需要对底层能力做深度重新对齐或知识注入 |
| **LoRA** | 冻结主模型，仅微调微小低秩矩阵 | LoRA 框架自带隔离机制 | 约 14 GB + 微末增量 | 消费级多卡/较好显卡单卡；特定垂直领域微调、风格模仿 |
| **QLoRA** | 主模型 4-bit 量化加载，再叠加上 LoRA | Bitsandbytes (量化组件) + LoRA | 约 4 GB + 微末增量 | 贫民窟玩家；快速验证概念模型；单张消费级显卡训中大模型 |

无论采用哪种微调方式，善用 **Gradient Checkpointing（重计算牺牲速度换显存）** 以及 **FlashAttention（优化注意力层的显存和速度）** 都可以让你的设备发挥远超预期的实力。
